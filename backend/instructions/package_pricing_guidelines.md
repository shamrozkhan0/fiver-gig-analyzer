# Packages & Pricing Guidelines

## Objective

Evaluate the seller's packages and pricing to determine whether they provide clear value, logical progression, and competitive positioning while remaining consistent with the gig's advertised service.

---

## Evaluate

Assess each package using the following criteria:

### Package Clarity

Determine whether the buyer can immediately understand:

* What is included
* What is delivered
* What makes the package valuable
* What differentiates it from the other packages

---

### Feature Completeness

Verify whether the package clearly defines relevant information such as:

* Deliverables
* Scope of work
* Revisions
* Delivery time (if provided)
* Included features

Recommend adding only information that is relevant to the seller's service.

---

### Package Differentiation

Each package should provide a meaningful increase in value.

A typical progression is:

* Basic → Essential service
* Standard → Expanded solution
* Premium → Most comprehensive solution

Packages should not appear nearly identical.

---

### Value for Money

Evaluate whether the included features justify the requested price.

Consider:

* Scope of work
* Deliverables
* Complexity
* Revisions
* Additional included value

Do not judge pricing by price alone.

---

### Pricing Progression

Verify that package prices increase logically as value increases.

Higher-priced packages should clearly offer additional benefits.

Poor progression includes:

* Nearly identical packages
* Large price differences with little additional value
* Lower packages offering almost everything available in higher tiers

---

### Delivery & Revisions

If delivery time or revisions are provided, determine whether they are appropriate for:

* Service complexity
* Package value
* Buyer expectations

Do not recommend unrealistic delivery times or unlimited revisions unless justified.

---

### Consistency

Packages should remain consistent with:

* Gig Title
* Gig Description
* Seller Expertise
* Category
* Tags

Packages must not advertise services that are not offered elsewhere in the gig.

---

## Pricing Recommendations

Recommend pricing changes only when supported by clear evidence.

Examples include:

* Features do not justify the price.
* Packages are difficult to distinguish.
* Package progression is inconsistent.
* Value offered is significantly above or below the requested price.

When recommending pricing adjustments, explain:

* What should change.
* Why the current pricing is unbalanced.
* How the recommendation improves buyer value.

---

## Optimization Strategy

Prefer improving existing packages rather than redesigning them.

Only recommend major structural changes when:

* Packages overlap significantly.
* Features are unclear.
* Value progression is weak.
* Pricing is unsupported by the included features.

---

## Success Criteria

A well-optimized package structure should:

* Clearly communicate value.
* Differentiate each package.
* Justify pricing through included features.
* Progress logically from Basic to Premium.
* Remain consistent with the rest of the gig.
* Help buyers confidently choose the package that fits their needs.
