# Tags Guidelines

## Objective

Evaluate and optimize gig tags to maximize discoverability while accurately representing the seller's service and buyer search intent.

---

## Evaluate

Assess the tag set for:

* Service relevance
* Buyer search intent
* Primary keyword coverage
* Secondary keyword coverage
* Keyword diversity
* Consistency with the gig
* Overall search coverage

---

## Relevance

Every tag should directly relate to the seller's actual service.

A strong tag should:

* Represent the offered service.
* Match the gig title.
* Support the description.
* Align with the seller's expertise.
* Reflect likely buyer searches.

Remove tags that describe services the seller does not offer.

---

## Keyword Coverage

The tag set should collectively represent:

* Primary service
* Supporting services
* Technologies or tools (when applicable)
* Relevant niches or industries (when applicable)

Avoid using multiple tags that target the same search intent.

Prioritize broader keyword coverage over repetitive variations.

---

## Tag Quality

Good tags are:

* Relevant
* Specific
* Buyer-focused
* Search-intent aligned
* Consistent with the gig

Avoid tags that are:

* Generic
* Misleading
* Irrelevant
* Repetitive
* Keyword stuffed

---

## Optimization Strategy

Preserve strong existing tags whenever possible.

Replace only tags that:

* Are irrelevant
* Duplicate the intent of other tags
* Target the wrong service
* Contribute little search value

Generate an entirely new tag set only when the existing tags provide poor overall coverage or fail to represent the seller's service accurately.

---

## Success Criteria

An optimized tag set should:
* Clearly represent the seller's primary service.
* Cover important supporting keywords without redundancy.
* Align with the title, description, expertise, packages, and category.
* Improve search coverage while remaining highly relevant.
