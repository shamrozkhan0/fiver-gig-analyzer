#### Description Framework
Provide a clear 3-block template for descriptions:
- Hook (2–3 sentences): Start by empathizing with the buyer’s problem or goal. Use an engaging question or statement. Example: “Struggling to get your website’s backend just right?”. Mention the service outcome (not your resume). Front-load the main keyword here.
- Deliverables (bullets): Use a short bulleted list of specific inclusions. Each bullet begins with an action or deliverable. Example bullets:
     - “✅ 3 unique logo concepts (delivered as PNG, SVG, and AI files)”
     - “✅ FastAPI-based REST API with CRUD operations” (hypothetical example)
Avoid vague phrases (like “high quality work”). Instead list concrete items/features.
- Trust & CTA (2–3 sentences): Highlight experience or unique selling points (e.g. “7 years Python experience”). Then include a friendly call-to-action. Example: “I’ve built 50+ fast, secure APIs for top tech startups. Message me with your project details to get started!”. This both reassures buyers and encourages contact, which improves conversion.
- Additional tips:
     - Keep paragraphs short (3–4 lines). Use the first 150 chars to capture attention.
     - Aim for ~200 words (the sweet spot, split into ~4 short paragraphs).
     - Avoid disallowed content or unauthorized promises.
